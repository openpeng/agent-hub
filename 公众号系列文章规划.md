## 产品解读篇（3篇）

### 第1篇：产品定位与愿景

**标题**：《AI Agent 生态的巴别塔困局，需要一个翻译官》

**定位**：开篇造势，用"巴别塔"类比切入 N×M 问题，建立 Agent Hub 的产品认知。

**核心内容**：AI 工具生态碎片化现状 → N×M 问题建模 → Agent Hub 的产品定位（跨平台互操作层）→ 三个产品模块概述（Market/Deploy/Builder）→ 目标用户画像 → 产品哲学（平台中立/渐进式采用/安全优先）

**文件**：`00-Agent-Hub-产品定位与愿景.md`

---

### 第2篇：核心功能全景图

**标题**：《Agent Hub 核心功能全景图：从创建到分发到执行》

**定位**：功能总览，让读者全面了解 Agent Hub 的能力边界。

**核心内容**：四层架构图（创建→分发→部署→执行）→ Market 五种资源类型详解 → Deploy 部署/导入/MCP Server 三大能力 → Builder 创建流程和 AI 辅助 → Runtime Pipeline 和子 Agent 协作 → 四个典型使用场景

**文件**：`00-Agent-Hub-核心功能全景图.md`

---

### 第3篇：快速上手指南

**标题**：《5 分钟上手 Agent Hub：从安装到创建你的第一个 Agent》

**定位**：实操教程，降低尝试门槛，引导读者动手。

**核心内容**：安装 agent-deploy → init 向导创建第一个 Agent → deploy 部署到 AI 工具 → use 从 Market 安装 → import 导入已有配置 → run 运行 Pipeline → Builder 图形界面创建 → 常用命令速查表

**文件**：`00-Agent-Hub-快速上手指南.md`

---

## 技术深度篇（6篇）

### 第4篇：项目架构总览

**标题**：《Agent Hub：用一份 Schema 统治 9 个 AI 编码工具的 Agent 生态》

**定位**：技术开篇，从工程视角建立整体认知。

**核心内容**：N×M 问题的工程建模 → 五模块架构（protocol/builder/deploy/market/compose）→ 三种 Schema 覆盖范围 → 各模块工程规模 → 数据流方向

**文件**：`01-Agent-Hub-正式发布.md`

---

### 第5篇：协议规范

**标题**：《agent.json v3.0 协议深度解析：subagent 模型与 4 层 fallback 策略》

**定位**：协议层深度解析。

**核心内容**：v2→v3 架构演进 → subagent 模型 → identity 工程约束 → instructions 三种格式 → 4 层 fallback 指令加载 → dependencies 运行时依赖 → team.json 三种协作模式 → workflow.json 6 种步骤类型 → JSON vs YAML 选型讨论

**文件**：`02-agent-json-设计哲学.md`

---

### 第6篇：适配引擎

**标题**：《adapt.ts 适配引擎：策略模式与 Pipeline-to-Prompt 翻译》

**定位**：跨平台适配的技术实现。

**核心内容**：适配问题的三个工程挑战 → 策略模式 9 平台分支 → 输出路径映射 → 内容格式差异 → Pipeline-to-Prompt 翻译 → 4 层 fallback + 文件级 fallback → 7 个导入适配器 → AdaptationResult 数据结构 → 设计模式总结

**文件**：`04-跨平台部署机制解析.md`

---

### 第7篇：构建器工程

**标题**：《agent-builder 工程解析：浏览器端打包、AI 配置生成与三实体 Market API》

**定位**：前端构建器的技术实现。

**核心内容**：两层路由架构 → 浏览器端 POSIX tar 手动构建 → agent.json 序列化器映射逻辑 → AI 配置生成与 sanitizeAIConfig 安全清洗 → Market API 服务层三实体封装 → Market 导入流程 → Zustand 状态管理

**文件**：`03-可视化构建器上手体验.md`

---

### 第8篇：运行时架构

**标题**：《Pipeline 运行时架构：步骤调度、模板变量与 OpenTelemetry 集成》

**定位**：执行引擎的技术深度。

**核心内容**：PipelineEngine 设计定位 → execute 主循环 → 条件执行与 invoke 语法糖 → TemplateResolver 变量解析 → Promise.allSettled 并行调度 → 错误处理与指数退避重试 → ToolRegistry 原型链 → Docker 沙箱隔离 → OpenTelemetry 集成 → AgentExecutor 外观模式

**文件**：`05-Pipeline-引擎与运行时执行.md`

---

### 第9篇：Market 后端工程

**标题**：《Market 后端工程：SQLite WAL、三阶段安全扫描与滑动窗口限流》

**定位**：Market 后端的技术实现。

**核心内容**：FastAPI 中间件链 → SQLite WAL 模式 → 11 张表同构设计 → 评分聚合与关联表同步 → 三阶段安全扫描 → API Key CSPRNG+SHA-256+常量时间比对 → 滑动窗口限流 → 流式下载与元数据提取 → 统一搜索

**文件**：`06-Agent-Market-市场生态.md`
