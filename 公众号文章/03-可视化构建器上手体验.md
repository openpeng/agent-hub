# agent-builder 工程解析：浏览器端打包、AI 配置生成与三实体 Market API

> 跨平台 AI Agent 互操作系统深度解析系列（三）

---

## 架构概览

agent-builder 是一个 React SPA，承担 Agent/Team/Workflow 的可视化创建、AI 辅助配置生成和 Market 发布。技术栈选型：React 19 + TypeScript 6 + Vite 8 + Zustand 5 + React Router 7。

路由采用两层嵌套结构：

```
/ (HomePage)              ← 独立页面，无布局
/docs (DocsPage)          ← 独立页面，无布局
/* (BuilderLayout)        ← 统一布局包裹
  /quick-start           ← 快速开始
  /intro                 ← Agent 配置（Step 1）
  /skills                ← Skill 选择（Step 2）
  /mcp-tools             ← MCP 工具配置（Step 3）
  /preview-publish       ← 预览与发布（Step 4）
  /market                ← Market 浏览与导入
```

`StepSync` 组件监听 `location.pathname` 变化，通过 `pathStepMap` 映射表将 URL 同步到 Zustand Store 的 `currentStep` 状态。这意味着浏览器的前进/后退按钮与构建步骤双向绑定——用户可以通过 URL 直接跳转到任意步骤。

## 浏览器端 tar.gz 打包

这是构建器中最有技术含量的部分。`packageBuilder.ts` 在浏览器端完成 tar.gz 打包，无需后端参与。

### POSIX tar 格式手动构建

```typescript
function writeTarHeader(name: string, size: number): Buffer {
  const header = Buffer.alloc(512);
  // name: 0-99 bytes, null-terminated
  header.write(name, 0);
  // mode: 100-107 bytes, octal string
  header.write('0000644\0', 100);
  // size: 124-135 bytes, octal string
  header.write(size.toString(8).padStart(8, '0') + '\0', 124);
  // typeflag: 156 bytes, '0' = regular file
  header[156] = 0x30; // '0'
  // magic: 257-263 bytes, "ustar\0"
  header.write('ustar\0', 257);
  // version: 264-265 bytes, "00"
  header.write('00', 264);
  // checksum: 148-155 bytes, computed after filling all other fields
  let sum = 0;
  for (let i = 0; i < 512; i++) sum += header[i];
  header.write(sum.toString(8).padStart(6, '0') + '\0 ', 148);
  return header;
}
```

每个文件打包为：512 字节 header + 文件数据 + 512 字节对齐填充。包末尾追加 1024 字节结束标记。整个 tar 二进制通过 `CompressionStream('gzip')` API 压缩为 `.tar.gz`。

### 打包内容

```typescript
async function buildAgentPackage(config: AgentConfig): Promise<PackageResult> {
  const agentJson = serializeAgentConfigToString(config);
  const readme = generateReadme(config);
  // 手动构建 tar → gzip 压缩 → 返回 Blob
  return { blob: tarGzBlob, agentJsonContent: agentJson };
}
```

包内容仅包含 `agent.json` 和自动生成的 `README.md`。Skills 和 MCP 工具配置内联在 agent.json 中，不作为独立文件打包。

## agent.json 序列化器

`agentJsonSerializer.ts` 将前端 Store 的 `AgentConfig` 转换为 Market 兼容的 agent.json。映射逻辑：

```
前端 Store (AgentConfig)         →  agent.json
─────────────────────────────────────────────
name (用户输入)                    →  identity.name (自动转 kebab-case)
version                           →  identity.version
name (原始)                       →  identity.display_name
description.summary               →  identity.description
developer                         →  identity.author
categories → categoriesToTags()   →  identity.tags (中文→英文映射)
categories[0]                      →  category (顶层字段)
description.detail                →  instructions.content
skills[] → skillRefToJson()       →  skills[] (Format B)
mcpTools[] → mcpToolRefToJson()   →  mcp_servers[] (Format A1)
```

中文分类到英文标签的映射：

| 中文 | 英文 |
|------|------|
| 办公效率 | productivity |
| 开发工具 | development |
| 数据分析 | data-analysis |
| 内容创作 | content-creation |
| 客户服务 | customer-service |

值得注意的是，当前序列化器输出的是 v2 兼容格式，不包含 `schema_version`、`entry`、`subagents` 等 v3.0 必填字段。这意味着通过构建器创建的 Agent 使用 v2 格式，运行时由 V2CompatibilityLayer 处理。

## AI 配置生成与清洗

AI 自动填写通过本地 agent-bridge 服务（`localhost:3210`）调用 LLM：

```typescript
// 1. 检查 Bridge 健康状态
const health = await fetch('http://localhost:3210/api/health', { signal: AbortSignal.timeout(3000) });

// 2. 发送生成请求
const result = await fetch('http://localhost:3210/api/generate-config', {
  method: 'POST',
  body: JSON.stringify({ userRequest: description, sessionId })
});

// 3. 清洗 AI 返回的原始配置
const sanitized = sanitizeAIConfig(rawConfig);
```

`sanitizeAIConfig()` 是关键的安全层。AI 模型返回的 JSON 不可信，必须经过严格清洗：

- `name`：转小写，正则白名单 `[a-zA-Z0-9\u4e00-\u9fff\-]+`（允许中文字符）
- `icon`：白名单校验，仅允许 8 个预定义 emoji
- `categories`：白名单校验，最多 3 个，映射为英文标签
- `skills`：数组映射为 `SkillRef[]`，缺失字段自动补全默认值
- `mcpTools`：数组映射为 `McpToolRef[]`，从 `tool.config` 提取 command/args/env
- `description`：拆分为 summary/detail/examples 三层结构
- `sampleInputs`：截断为最多 5 条字符串

清洗后的配置通过预览面板展示（名称、图标、简介、分类、Skills 数量、MCP 工具数量、欢迎语、输入示例），用户确认后应用到 Zustand Store。

## Market API 服务层

`marketApi.ts` 封装了与 Market 后端的全部交互，支持 Agent/Team/Workflow 三种实体：

```typescript
// 基础配置
const MARKET_URL = import.meta.env.VITE_MARKET_URL || 'http://localhost:8321';
const getAuthHeaders = () => ({
  'Authorization': `Bearer ${localStorage.getItem('market_api_key')}`
});
```

API 端点遵循 RESTful 设计，三种实体共享统一的 URL 模式：

```
GET    /api/v1/{entity}?q=&category=&tags=&page=&page_size=   ← 搜索
GET    /api/v1/{entity}/{id}                                  ← 详情
POST   /api/v1/{entity}/{id}/ratings                          ← 评分
DELETE /api/v1/{entity}/{id}                                  ← 删除
POST   /api/v1/{entity} (FormData: file)                      ← 上传
```

`{entity}` 可以是 `agents`、`teams` 或 `workflows`。返回数据结构也完全同构：`ListItem`（id, display_name, version, description, category, tags, download_count, rating）和 `Detail`（继承 ListItem + readme, json_content, dependencies 等）。

上传使用 `FormData`，文件以 `file` 字段名提交，支持 `force` 参数覆盖已有版本。搜索支持 `q`（关键词）、`category`（精确匹配）、`tags`（逗号分隔）、`sort`（name/rating/download_count/created_at）、`order`（asc/desc）等参数。

## Market 导入流程

MarketPage 组件实现了从 Market 导入已有 Agent 的功能：

1. 用户在 Market 搜索并选择目标 Agent
2. 调用 `GET /api/v1/agents/{id}` 获取详情（包含 `json_content`）
3. 解析 `json_content` 为 agent.json 对象
4. 通过 `marketMapper.ts` 反向映射为前端 `AgentConfig` 格式
5. 填充到 Zustand Store，用户可在此基础上修改、扩展
6. 重新发布时保留原作品的署名信息

`tarGzParser.ts` 负责解析下载的 tar.gz 包，提取其中的 agent.json 和 README.md。

## 状态管理设计

Zustand Store 管理整个构建流程的状态：

```typescript
interface AgentConfig {
  name: string;
  version: string;
  icon: string;
  description: { summary: string; detail: string; examples: string[] };
  developer: string;
  categories: string[];
  skills: SkillRef[];
  mcpTools: McpToolRef[];
  welcomeMessage: string;
  sampleInputs: string[];
  // ...
}
```

Store 采用扁平结构，所有字段在同一层级。Skills 和 MCP Tools 使用数组存储，支持拖拽排序（@dnd-kit）。`currentStep` 字段与 URL 路由双向同步，`StepSync` 组件在路由变化时自动更新。

## 技术选型的工程考量

| 选型 | 理由 |
|------|------|
| React 19 | 并发渲染 + use() hook，为复杂表单场景提供更好的性能 |
| TypeScript 6 | 严格类型检查，减少运行时错误 |
| Vite 8 | ESM 原生支持，HMR 毫秒级响应 |
| Zustand 5 | 比 Redux 轻量 10x，无 boilerplate，适合中等复杂度状态 |
| React Router 7 | 嵌套路由 + 数据加载，支持 URL 驱动状态 |
| @dnd-kit | 声明式拖拽 API，支持 Sortable 和 Grid 布局 |
